The `Icon` component allows you to include [Font Awesome](https://fontawesome.com/icons) icons directly in your documentation. Icons can be customized with different styles, colors, and sizes.

## Basic Usage

Use the `Icon` component with a `name` prop to display an icon:

<Tabs>
  <Tab title="Example">
    <Icon name="rocket" /> Launch
    <br/>
    <Icon name="check" color="#22C55E" /> Success
    <br/>
    <Icon name="warning" size="lg" /> Warning
  </Tab>
  <Tab title="Markdown">
    ```jsx
    <Icon name="rocket" /> Launch
    <Icon name="check" color="#22C55E" /> Success
    <Icon name="warning" size="lg" /> Warning
    ```
  </Tab>
</Tabs>

## Configuration

The Icon component accepts the following props:

| Prop | Type | Required | Description |
|------|------|----------|-------------|
| `name` | string | Yes | Name of the Font Awesome icon |
| `color` | string | No | Custom color (hex, RGB, or color name) |
| `size` | string | No | Size variant ('xs', 'sm', 'lg', 'xl', '2x', etc.) |
| `style` | string | No | Icon style ('regular', 'solid', 'brands') |

## Best Practices

- Use icons consistently throughout your documentation
- Choose appropriate sizes that match your content's scale
- Ensure icon colors maintain sufficient contrast with the background
- Consider using regular (outlined) style for less emphasis and solid style for more emphasis
