## Enabling Analytics

Most analytics integrations can be configured directly in `docs.yml`. You only need to include entries for the platforms you want to connect.

```yaml docs.yml
analytics:
  ga4:
    measurement-id: ${GA4_MEASUREMENT_ID}
  posthog:
    api-key: ${POSTHOG_API_KEY}
    endpoint: https://self.hosted.posthog.com/
  segment:
    write-key: ${SEGMENT_WRITE_KEY}
  intercom:
    app-id: ${INTERCOM_APP_ID}
    endpoint: https://intercom.custom-instance.com/
  fullstory:
    org-id: ${FULLSTORY_ORG_ID}
```

### Environment Variables

If your docs configuration is public, then we do not advise adding secret values directly to `docs.yml`.
Instead, you can reference an environment variable by using the syntax `${VARIABLE_NAME}`.

<Note>
If you are using GitHub Workflows to trigger docs generation, you must make sure that the environment variables are available during the workflow run.

```yaml {4,5}
- name: Publish Docs
  env:
    FERN_TOKEN: ${{ secrets.FERN_TOKEN }}
    GA4_MEASUREMENT_ID: ${{ secrets.GA4_MEASUREMENT_ID }}
    POSTHOG_API_KEY: ${{ secrets.POSTHOG_PROJECT_API_KEY }}
  run: |
    npm install -g fern-api
    fern generate --docs
```

</Note>
